## Abstract
The **Web Dev Agent Workflow** is a sophisticated, automated system designed to streamline the end-to-end process of web development. By leveraging a multi-agent architecture, the system coordinates project management, high-end frontend development, and automated GitHub repository management. The workflow is specifically optimized for creating elegant, responsive, and emotional single-page applications (SPAs) using modern technologies like **React** and **Vite**. It integrates advanced tools for real-time data handling, such as Google Apps Script, and ensures a clean, modular codebase without the overhead of utility-first frameworks.

## Workflow Architecture
The system operates through three primary intelligent agents:
1.  **AI Employee (Project Manager)**: Orchestrates the entire workflow, manages the task list, and delegates technical responsibilities to specialized agents.
2.  **Coding Agent (Senior Developer)**: Responsible for writing modular JSX and CSS. It handles environment setup, dependency management (npm), and local testing.
3.  **Web Agent (DevOps & GitHub Manager)**: Manages the remote repository lifecycle, including cleaning, initializing, and force-pushing the codebase to ensure a pristine deployment state.

## Precautions
*   **Tooling Constraints**: The Coding Agent is strictly prohibited from adding new tools via the `agent-builder`; any required tools must be manually added by the user.
*   **Framework Limitations**: The workflow is optimized for React/Vite using pure CSS. It explicitly avoids utility-first frameworks like TailwindCSS or PostCSS.
*   **Dependency Management**: `npm install` should only be executed once after all code files are written to prevent environment conflicts.
*   **GitHub Token Safety**: Ensure the GitHub Personal Access Token (PAT) has the correct 'Contents' (Read & Write) permissions for the target repository.

**Q: Does the system handle backend development?**
**A:** The workflow primarily focuses on high-end frontend development. However, it integrates with serverless solutions like Google Apps Script and external APIs to handle dynamic data and form submissions.

**Q: Can I use TailwindCSS with this workflow?**
**A:** No, the system is intentionally designed to use pure CSS and modular JSX to maintain a clean, lightweight, and high-performance codebase, avoiding the overhead of utility-first frameworks.

**Q: How is deployment handled?**
